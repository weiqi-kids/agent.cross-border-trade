# 品牌定位文件

## 1. 核心資訊

| 項目 | 內容 |
|------|------|
| **網站名稱** | 全球貿易與供需智慧分析系統 (Global Trade Intelligence System) |
| **一句話描述** | 追蹤六大經濟體（台美中日韓德）貿易動態與出口管制政策的自動化情報分析平台 |
| **核心價值主張** | 整合多源官方數據（UN Comtrade、US Census、World Bank、中國商務部），透過 AI 自動化擷取、NLP 分析與結構化報告，提供即時、可信、可追溯的貿易情報，幫助專業人士快速掌握全球貿易變化與政策風險 |

## 2. 目標受眾

### 主要受眾 (Primary)

| 項目 | 描述 |
|------|------|
| **是誰** | 貿易數據分析師、供應鏈管理者、政策研究者 |
| **目標/需求** | 快速掌握貿易流量變化、市場集中度風險、出口管制政策演進，支援決策與風險評估 |
| **痛點** | (1) 數據源分散（UN、Census、World Bank 等需逐一查詢）；(2) 政策文件海量且語言多元（中英夾雜）；(3) 缺乏結構化整合與趨勢分析；(4) 時效性不足，依賴人工追蹤耗時長 |
| **使用場景** | 週一準備週報、月初供應鏈風險評估、政策發布後快速解讀、客戶詢問特定國家貿易數據時查閱 |

### 次要受眾 (Secondary)

| 項目 | 描述 |
|------|------|
| **是誰** | 投資分析師、跨境貿易業者、財經媒體 |
| **目標/需求** | 投資分析師：產業趨勢判斷、供應鏈重組投資主題；貿易業者：合規指引、行動清單；財經媒體：可引用數據、新聞角度 |
| **痛點** | 投資分析師：缺乏季度級別的趨勢總結；貿易業者：法規文件艱澀難懂；媒體：數據來源權威性驗證耗時 |
| **使用場景** | 投資分析師：季度投資報告撰寫；貿易業者：新政策發布後確認合規影響；媒體：突發貿易事件（如美台逆差激增）時查找數據支持 |

## 3. 差異化定位

| 項目 | 內容 |
|------|------|
| **與競品的差異** | (1) **自動化程度**：AI 驅動的 fetch-extract-analyze-report 全流程（vs. 傳統人工編輯）；(2) **數據可追溯性**：每個數據點標註來源檔案與擷取時間（vs. 一般財經媒體無出處）；(3) **結構化輸出**：符合 Schema.org 標準、含 JSON-LD、AEO 標記（vs. PDF 報告或純文字內容）；(4) **雙語整合**：中英文政策文件一站式彙整（vs. 單語種平台） |
| **獨特優勢** | (1) **技術透明度**：開源架構（GitHub 公開），使用者可審核數據來源與處理邏輯；(2) **頻率多元**：週報（trade_briefing）、雙週報（compliance_digest）、月報（supply_chain/policy_tracker）、季報（investment_insight）滿足不同時效需求；(3) **角色導航**：首頁依使用者角色（分析師/管理者/研究者等）推薦適合報告，降低學習成本 |
| **不做什麼** | (1) 不提供 B2C 投資建議或股票推薦；(2) 不涵蓋六大經濟體以外的國家；(3) 不處理即時新聞快訊（專注於結構化數據分析）；(4) 不提供付費訂閱或客製化報告（當前為開放存取） |

## 6. 改版目標

| 優先級 | 目標 | 說明 |
|--------|------|------|
| **P0** | 完善 SEO + AEO 標籤系統 | 所有報告必須包含：(1) 完整 Meta 標籤（title/description/OG）；(2) 5 種必填 Schema（WebPage/Article/Person/Organization/BreadcrumbList）；(3) SGE/AEO 標記（.key-answer/.key-takeaway/.data-highlight）。**理由**：當前 SEO 基礎已建立（見 W08 報告），需系統化確保所有頁面達標 |
| **P0** | 改善首頁資訊架構 | (1) 強化角色導航表格（加入使用情境說明）；(2) 簡化「本週重點」區塊（當前 emoji 指標缺乏圖例說明）；(3) 新增「如何使用本站」快速入門區塊。**理由**：降低新訪客學習成本，提升轉換率 |
| **P0** | 確保數據來源標示一致性 | 所有報告必須明確標註：(1) 數據來源（UN Comtrade/US Census/etc.）；(2) 數據時間點（如「2025 全年」「2026-02-17 擷取」）；(3) 自動化程度（如「80% 自動化」）。**理由**：強化品牌核心價值「可追溯性」 |
| **P1** | 優化 Core Web Vitals | 改善 LCP（首屏載入速度）與 CLS（版面穩定性），目標達 PageSpeed Insights 綠燈。**方法**：(1) 優化 Mermaid 圖表載入；(2) 圖片 lazy loading；(3) Jekyll 靜態資源壓縮 |
| **P1** | 新增「數據更新時間軸」頁面 | 展示各 Layer（bilateral_trade_flows/cn_export_control 等）的最後更新時間與下次預計更新時間。**理由**：建立信任感，讓使用者理解數據時效性 |
| **P1** | 增加內部連結密度 | 在報告中增加歷史報告連結（如「上週報告指出...」「詳見 2026-02 供應鏈月報」）。**理由**：提升 SEO、降低跳出率、引導深度閱讀 |
| **P2** | 建立 FAQ 頁面 | 回答常見問題（如「數據來源是否可靠」「為何只涵蓋六國」「更新頻率為何」）。**理由**：減少使用者疑慮，可作為 FAQ Schema 的內容來源 |
| **P2** | 新增 RSS/訂閱功能 | 提供 RSS feed 或 Email 訂閱（透過 GitHub Actions 自動推送）。**理由**：提升回訪率，建立專業受眾關係 |
| **P2** | 建立「關於本站」頁面 | 說明系統架構、AI 角色、開源理念。**理由**：當前「Powered by Claude Code」僅有 GitHub 連結，缺乏系統性介紹，不利建立信任 |
