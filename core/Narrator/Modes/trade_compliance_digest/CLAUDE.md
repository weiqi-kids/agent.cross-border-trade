# trade_compliance_digest — 貿易合規摘要 Mode

## Mode 定義表

| 項目 | 內容 |
|------|------|
| **Mode name** | trade_compliance_digest（貿易合規摘要） |
| **Purpose and audience** | 為跨境貿易業者、報關行、物流公司提供簡明合規指引與實務注意事項 |
| **Source layers** | cn_export_control（主要）、us_trade_census（美國市場） |
| **Automation ratio** | 60% — 政策彙整自動化，合規建議需人工審核 |
| **Content risk** | 合規建議誤導、法律解讀偏差、未及時更新政策變化 |
| **Reviewer persona** | 法規與責任審核員、使用者誤導風險審核員 |

## 資料來源定義

### 主要來源（docs/ 檔案）

1. **cn_export_control** — 讀取 `docs/Extractor/cn_export_control/` 下所有 category 的 `.md` 檔
   - `regulation_update/` — 法規更新（重點：即將生效）
   - `enforcement_action/` — 執法行動（重點：近期案例）
   - `controlled_item_change/` — 管制物項變更
   - `policy_guidance/` — 政策指導（重點：實務 Q&A）

2. **us_trade_census** — 讀取 `docs/Extractor/us_trade_census/` 下的 `.md` 檔（美國市場注意事項）

### 輔助來源（Qdrant 語意搜尋）

透過 Qdrant 搜尋相關歷史違規案例，提供警示。

### 歷史參考策略

#### 短期趨勢（前 4 期）

1. 讀取 `docs/Narrator/trade_compliance_digest/` 下前 4 期報告
2. 僅提取「一、即將生效的政策」區塊
3. 追蹤已提醒但尚未生效的政策

**整合方式**：
- 標註「本期新增」vs「持續追蹤」
- 移除已過期的政策提醒

#### 語意搜尋（Qdrant）

**查詢設計**：基於本期管制物項生成 3-5 個查詢

| 查詢主題 | 查詢範例 | 目標 |
|---------|---------|------|
| 違規案例 | 「{管制物項}違規處罰」 | 警示案例 |
| 合規指引 | 「{產業}出口合規」 | 實務指引 |
| 執法趨勢 | 「不可靠實體清單」 | 執法動態 |

**搜尋參數**：
- 每個查詢返回 3 筆結果
- 總計不超過 10 筆（去重後）
- 僅包含 score > 0.5 的結果

**整合方式**：
- 在「二、執法動態」引用相關歷史案例
- 正文引用時標註來源和 Qdrant score
