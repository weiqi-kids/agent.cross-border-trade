# policy_tracker — 出口管制政策追蹤 Mode

## Mode 定義表

| 項目 | 內容 |
|------|------|
| **Mode name** | policy_tracker（出口管制政策追蹤） |
| **Purpose and audience** | 為政策研究者提供深度政策追蹤與歷史脈絡；為貿易合規人員提供政策彙整與風險警示 |
| **Source layers** | cn_export_control（主要）、bilateral_trade_flows（貿易影響）、us_trade_census（美國視角） |
| **Automation ratio** | 50% — 政策彙整自動化，影響評估與合規建議需人工審核 |
| **Content risk** | 法律解讀偏差、合規建議可能誤導、政策影響過度推論 |
| **Reviewer persona** | 法規與責任審核員、政策解讀審核員、使用者誤導風險審核員 |

## 資料來源定義

### 主要來源（docs/ 檔案）

1. **cn_export_control** — 讀取 `docs/Extractor/cn_export_control/` 下所有 category 的 `.md` 檔
   - `regulation_update/` — 法規更新
   - `enforcement_action/` — 執法行動
   - `controlled_item_change/` — 管制物項變更
   - `policy_guidance/` — 政策指導
   - `international_cooperation/` — 國際合作

2. **bilateral_trade_flows** — 讀取 `docs/Extractor/bilateral_trade_flows/` 下的 `.md` 檔（用於評估貿易影響）

3. **us_trade_census** — 讀取 `docs/Extractor/us_trade_census/` 下的 `.md` 檔（用於美國視角分析）

### 輔助來源（Qdrant 語意搜尋）

透過 Qdrant 搜尋相關歷史政策，建立政策演進脈絡。

### 歷史參考策略

#### 短期趨勢（前 3 個月）

1. 讀取 `docs/Narrator/policy_tracker/` 下前 3 個月報告
2. 僅提取「執行摘要」和「一、政策概覽」區塊
3. 檔案不存在時跳過，在「已知限制」中說明

**整合方式**：
- 統計表格加入「上月」「3 月平均」欄位
- 標註持續追蹤的政策項目

#### 去年同期（同月）

1. 查找 `{YYYY-1}-{MM}-policy-tracker.md` 報告
2. 用於同比分析（政策發布頻率、執法趨勢）

**整合方式**：
- 在「政策概覽」加入 YoY 欄位
- 無去年資料時標註「N/A（無歷史資料）」

#### 語意搜尋（Qdrant）

**查詢設計**：基於本期重點政策動態生成 5-8 個查詢

| 查詢主題 | 查詢範例 | 目標 |
|---------|---------|------|
| 政策脈絡 | 基於本期政策關鍵詞（如「稀土」「半導體」） | 追溯歷史相關政策 |
| 執法趨勢 | 「不可靠實體清單」「出口違規案例」 | 執法案例對比 |
| 國際協調 | 「中歐出口管制對話」「安世半導體」 | 國際合作脈絡 |
| 產業影響 | 「{管制物項}供應鏈影響」 | 產業面分析 |

**搜尋參數**：
- 每個查詢返回 5 筆結果
- 總計不超過 15 筆（去重後）
- 包含 score > 0.45 的結果（需更廣泛脈絡）

**整合方式**：
- 在「二、深度分析」引用相關歷史政策
- 正文引用時標註來源和 Qdrant score
- 建立政策演進時間軸
